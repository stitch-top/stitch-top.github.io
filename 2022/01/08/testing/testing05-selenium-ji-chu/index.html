Selenium 基础

Selenium提供了一个简单的API，使用WebDriver来编写校验测试，通过API可直观地使用Selenium WebDriver的所有功能。

1 概念

  • 概念
    • 官方文档，PyPI的官方库支持下载Selenium，命令安装：pip install selenium。
    • 各浏览器驱动
      • Firefox：https://github.com/mozilla/geckodriver/releases。
      • Chrome：https://chromedriver.storage.googleapis.com/index.html。
      • Edge：https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver。
      • 下载解压驱动，将其复制到Python的安装目录中，与python.exe同一文件夹位置。
    • 使用Edge驱动报错
      • 报错信息：Message: 'MicrosoftWebDriver.exe' executable needs to be in Path。
      • 驱动版本正确，路径也已加，则将驱动名msedgedriver.exe改为MicrosoftWebDriver.exe。

2 定位元素

  • 定位元素
    • 定位单个元素：公共方法。
      • find_element_by_id、find_element_by_name。
      • find_element_by_xpath、find_element_by_tag_name。
      • find_element_by_class_name、find_element_by_css_selector。
      • find_element_by_link_text、find_element_by_partial_link_text。
    • 定位一组元素：返回一个List列表，公共方法。
      • find_elements_by_id、find_elements_by_name。
      • find_elements_by_xpath、find_elements_by_tag_name。
      • find_elements_by_class_name、find_elements_by_css_selector。
      • find_elements_by_link_text、find_elements_by_partial_link_text。
    • find_element和find_elements：私有方法。
      • 调用这两个私有方法，通过By声明定位，传入对应定位方法的定位参数。
      • 使用前先导入模块：from selenium.webdriver.common.by import By。
      • 两参数分别是定位类型和定位值：driver.find_element(By.ID, 'kw')。
        • ID = "id"、NAME = "name"。
        • XPATH = "xpath"、TAG_NAME = "tag name"。
        • CLASS_NAME = "class name"、CSS_SELECTOR = "css selector"。
        • LINK_TEXT = "link text"、PARTIAL_LINK_TEXT = "partial link text"。

2-3 XPath

  • XPath
    • 注意：该定位方式在Selenium版本4.x.x已弃用，若要使用，需回退到4.0.0之前的版本。
    • 索引，当前层级下第1个input标签：find_element_by_xpath("//input[1]")。
    • 元素属性定位
      • 特定属性名@属性名：find_element_by_xpath("//form[@id]")。
      • @属性名="属性值"：find_element_by_xpath("//form[@id='loginForm']")。
      • 任意属性名@*="属性值"：find_element_by_xpath("//input[@*='username']")。
      • 多属性结合
        • find_element_by_xpath("//input[@name='continue'][@type='submit']")。
        • find_element_by_xpath("//input[@name='continue' or @type='submit']")。
        • find_element_by_xpath("//input[@name='continue' and @type='submit']")。
    • 路径定位
      • 相对路径//：find_element_by_xpath("//form")。
      • 绝对路径/：find_element_by_xpath("/html/body/form")。
      • 当前路径.：find_element_by_xpath("//form/.")，等同于//form。
      • 上级路径..：find_element_by_xpath("//form/input[1]/..")，等同于//form。
    • 函数定位
      • last()：find_element_by_xpath("//input[last()]")。
      • text()：find_element_by_xpath("//a[text()='Mouclick']")。
      • contains()：find_element_by_xpath("//input[contains(@type, 'submit')]")。
      • starts-with()：find_element_by_xpath("//a[starts-with(text(), 'Mouclick')]")。
    • 轴定位
      • child：find_element_by_xpath("//form/child")。
      • parent：find_element_by_xpath("//input[1]/parent")。
      • preceding：find_element_by_xpath("//a[text()='Continue']/preceding::a")。
      • following：find_element_by_xpath("//a[text()='Continue']/following::input[1]")。

2-6 CSS选择器

  • CSS选择器
    • 标签名定位：find_element_by_css_selector("p")。
    • id定位：find_element_by_css_selector("#loginForm")。
    • class定位：find_element_by_css_selector(".content")。
    • 元素定位
      • element>element：find_element_by_css_selector("form>p")，儿子节点。
      • element element：find_element_by_css_selector("form p.content")，子孙节点。
      • element+element：find_element_by_css_selector("a+input")，紧挨着的兄弟节点，子节点不算。
      • 注意：使用Firefox浏览器时，+号前后要加空格，否则定位不到，Chrome和Edge不用，CSS本身也不用。
    • 属性定位
      • [属性=属性值]：find_element_by_css_selector("[name='username']")。
      • [属性^=属性值]：find_element_by_css_selector("[name^='con']")，name属性开头为con的标签。
      • [属性*=属性值]：find_element_by_css_selector("[name*='con']")，name属性包含con的所有标签。
      • [属性$=属性值]：find_element_by_css_selector("[name$='nue']")，name的属性值结尾是nue的所有标签。
    • 下标定位
      • #loginForm>:last-child：id为loginForm标签下的最后1个标签。
      • #loginForm>:nth-last-child(2)：id为loginForm标签下的倒数第2个标签。
      • #loginForm>input:nth-child(3)：id为loginForm的标签下的第3个标签，且标签名为input。
      • #loginForm>:nth-child(1)或#loginForm>:first-child：id为loginForm标签下的第1个标签。

3 等待设置

  • 等待设置
    • 固定等待：timd.sleep()。
      • 不管浏览器是否加载完，程序都得等待，对于调试阶段十分有用。
      • 但太过死板，严重影响程序执行速度，自动化测试时不建议使用。
    • 隐式等待：implicitly_wait()。
      • 规定时间内网页加载完，则执行下一步，否则等到时间截止，再执行下一步。
      • 该等待方式对于整个浏览器驱动的周期都起作用，所以只需要设置一次即可。
    • 显示等待：WebDriverWait()。
      • 程序隔几秒看一眼，条件成立则执行下一步，否则继续等待。
      • 直到超过设置的最长时间，然后抛出TimeoutException异常。

3-3 显示等待

  • 显示等待
    • WebDriverWait(driver, timeout, poll_frequency, ignored_exceptions)
    • 参数说明
      • driver：浏览器驱动。
      • timeout：最长超时时间，默认以秒为单位。
      • poll_frequency：调用until或until_not方法的间隔时间，默认为0.5秒。
      • ignored_exceptions：超时后的异常信息，默认抛出NoSuchElementException异常。
    • 调用方法
      • WebDriverWait(driver, timeout=3).until(method, message="")，返回值True。
      • WebDriverWait(driver, timeout=3).until_not(method, message="")，返回值False。
    • expected_conditions类提供的预期条件判断方法
      • alert_is_present：判断页面上是否存在alert。
      • element_to_be_clickable：判断某个元素是否可以点击。
      • title
        • title_is：判断当前页面的标题是否等于预期字符串。
        • title_contains：判断当前页面的标题是否包含预期。
      • presence
        • presence_of_element_located：验证只要一个符合条件的元素被加载出来，就可通过。
        • presence_of_all_elements_located：验证必须所有符合条件的元素都加载出来才通过。
      • staleness_of：判断一个元素是否仍在DOM中，传入WebElement，可判断页面是否刷新了。
      • visibility
        • visibility_of：判断元素是否可见，传入WebElement。
        • visibility_of_element_located：判断元素是否可见，传入的参数是元组类型的locator。
        • invisibility_of_element_located：判断元素是否不可见，传入参数是元组类型的locator。
      • text
        • text_to_be_present_in_element：判断某个元素中的text是否包含预期的字符串。
        • text_to_be_present_in_element_value：判断某个元素的value属性是否包含预期的字符串。
      • frame_to_be_available_and_switch_to_it：判断表单是否可以切入，可切入则返回True并切入。
      • element
        • element_to_be_selected：判断元素是否被选中，传入WebElement对象。
        • element_located_to_be_selected：判断元素是否被选中，传locator元组。
        • element_selection_state_to_be：判断元素是否被选中，传入WebElement对象和状态，相等返回True。
        • element_located_selection_state_to_be：判断元素是否被选中，传入locator以及状态，相等返回True。

6-2 下载文件

  • 下载文件
    • Firefox
      • browser.download.dir
        • 自定义下载路径，默认只自动创建一级目录，若指定了多级不存在的目录，会下载到默认路径中。
        • 例如：定义了多级不存在的目录G:\\Download\\Python，文件就会自动下载到默认下载目录中。
      • browser.download.folderList
        • 0表示下载到桌面，1表示下载到浏览器默认下载路径，2表示使用自定义下载路径。
        • 与browser.download.dir配合使用，如果设置成0或1，那么dir路径的配置将无用。
      • browser.download.manager.useWindow：设定下载框是否要进行隐藏。
      • browser.download.manager.closeWhenDone：设定下载结束后是否自动关闭下载框。
      • browser.download.manager.alertOnEXEOpen：设定下载.exe文件是否要弹出警告。
      • browser.download.manager.showWhenStarting：设定启动下载时是否显示下载管理器。
      • browser.download.manager.focusWhenStarting：设定开始下载时是否获取浏览器焦点。
      • browser.download.manager.showAlertOnComplete：设定下载结束后是否显示下载完成提示框。
      • browser.helperApps.alwaysAsk.force
        • 未知的MIME类型文件，浏览器会弹窗让用户处理，默认值为True。
        • 将默认值改为False，表示不会记录打开未知MIME类型文件的方式。
      • browser.helperApps.neverAsk.openFile

8 分页爬取B站数据

  • 分页爬取B站数据
    • 爬虫：即获取网页并提取和保存信息的自动化程序。
    • 获取网页源代码，分析源代码，从中提取想要的数据，最通用的方法是使用正则表达式。
    • 如果网页的结构存在一定的规则，还可以根据一些节点属性，结合定位元素来提取数据。
    • 提取数据后，一般都会将数据保存到文档中，例如：TXT文本、JSON文本、Excel文档。
    • 或保存到数据库中，如MySQL、MongoDB等，甚至借助SFTP等工具保存到远程服务器上，方便后续的使用。
